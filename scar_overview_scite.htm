SCAR Editor

We use SCitE to write our script files. SCitE is a free source code editor which handles LUA as well as a number of other languages. When you first run SCitE set the language to Lua in the Language dropdown (Ctrl+L).

SCitE - http://scintilla.sourceforge.net/SciTE.html

Features

Some features of SCitE we will be taking advantage of include:

API Auto Complete - If the function we are writing exists in the API a list box will appear when you start writing that function with all the possible endings. *NOTE - This feature is not immediately available but will be forthcoming.

User Defined Auto Complete - If you hit "Ctrl+Enter" SCitE will display a list of everything in the document you are working on that starts with what you are typing. For example if you type "r" then hit auto complete you might get a list that looks like this:

Function Query - As you write a function that is already in the API SCitE will show you what the function wants so you don't have to refer to a SCARDocto look it up. If this disappears and you want to get it back you can press "(" twice to see it again. *NOTE - This feature is not immediately available but will be forthcoming.

Highlighting and Coloring - ScitE will automatically highlight or color certain aspects of your script.

CTRL+F7 Compile - if you hit "CTRL+F7" SCitE will test compile your script and let you know certain formatting errors that would keep the game from running the script. It won't test game functions but will let you know if your script is missing things like "end"s or opening or closing brackets. Below is a good compile and a broken script.

F5 Compile - If you write a test script for logic with no game specific functions SCitE will compile it and show the output in the Output Window (F8). Be sure to save the file type as .lua. This is a good way to practice writing simple scripts or to do the tutorials on the LUA site.
